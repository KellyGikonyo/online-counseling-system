Online Counseling System

Project Report Documentation

Executive Summary

The Online Counseling System is a comprehensive web-based platform designed to facilitate communication and session management between students and counselors in an educational environment. The system provides secure authentication, real-time messaging, session booking, feedback mechanisms, and administrative oversight capabilities.

Project Type: Full-stack Web Application

Development Period: Academic Project

Target Users: Students, Counselors, and Administrators

1. Project Overview

1.1 Project Description

The Online Counseling System is a digital platform that bridges the gap between students seeking guidance and counselors providing support services. The system eliminates geographical barriers and provides a secure, efficient way for students to access counseling services remotely.

1.2 Project Objectives

  • Provide a secure platform for student-counselor communication
  • Enable efficient session booking and management
  • Implement real-time messaging capabilities
  • Establish feedback and rating systems
  • Provide comprehensive administrative oversight
  • Ensure data privacy and security

1.3 Target Audience

  • Students: Primary users seeking counseling services
  • Counselors: Professional counselors providing guidance
  • Administrators: System managers overseeing operations

2.3 Database Schema

The system uses SQLite3 database with the following main tables:

Users Table

CREATE TABLE users ( username TEXT PRIMARY KEY, password TEXT, role TEXT, name TEXT, email TEXT, active INTEGER DEFAULT 1 );

Sessions Table

CREATE TABLE sessions ( id INTEGER PRIMARY KEY AUTOINCREMENT, student TEXT, counselor TEXT, date TEXT, time TEXT, reason TEXT );

Feedback Table

CREATE TABLE feedback ( id INTEGER PRIMARY KEY AUTOINCREMENT, fromUser TEXT, toUser TEXT, rating INTEGER, comment TEXT );

Messages Table

CREATE TABLE messages ( id INTEGER PRIMARY KEY AUTOINCREMENT, threadId INTEGER, fromUser TEXT, toUser TEXT, text TEXT, time TEXT, read INTEGER DEFAULT 0, attachments TEXT );

6. Database Design & Management

6.1 Database Architecture

The system uses SQLite3 as the database engine, chosen for its:

  • Lightweight nature and zero-configuration setup
  • ACID compliance for data integrity
  • Single file storage for easy deployment
  • Excellent performance for small to medium applications

6.2 Data Relationships

  • Users → Sessions: One-to-many relationship between users and sessions
  • Users → Messages: Many-to-many relationship through message threads
  • Users → Feedback: Many-to-many relationship for rating system

6.3 Data Integrity

  • Primary key constraints on all tables
  • Foreign key relationships maintained through application logic
  • Soft delete implementation (active flag) for user management
  • Input validation and sanitization
